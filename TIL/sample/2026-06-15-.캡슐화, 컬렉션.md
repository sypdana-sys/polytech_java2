## 오늘 배운 내용: 캡슐화 (encapsulation)


- (왜)휴먼 에러 방지를 위해 캡슐화를 씀: 필드에서 개발자의 잘못된 접근에 대한 제어 방법 제공, 현실세계와 동잃하게 프로그래밍 하려는 제어 방법
- (언제)
- (무엇을)멤버와 클래스..접근 지정자 (access modifier): private(-), package private(), public(+) ..protected(~)잘안씀
- *tip* 멤버: 필드는 전부 private, 메소드는 전부 public, 클래스: 보통 public
- (어떻게)필드에 접근하기 위한 메소드인 getter와 setter
- (어디서) setter 내부에서 인수의 타당성 검사를 수행
- (누가) JAVA
